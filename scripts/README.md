# API 测试脚本使用说明

## 测试覆盖范围

### 1. 会话管理测试（2个）
- ✅ 创建新会话
- ✅ 恢复已有会话

### 2. 分步保存测试（5个）
- ✅ 保存第一步（性别）
- ✅ 保存第二步（目标）
- ✅ 保存完整数据（增量合并）
- ✅ 未授权访问应失败（401）
- ✅ 访问不存在的记录应失败（404）

### 3. 提交与计算测试（4个）
- ✅ 提交不完整数据应失败（422）
- ✅ 提交完整数据应成功
- ✅ 重复提交应返回已有结果（幂等性）
- ✅ 提交后禁止修改 stepData（403）

### 4. 边界值测试（5个）
- ✅ 体重边界值 - 最小值 20kg
- ✅ 体重边界值 - 最大值 300kg
- ✅ 身高边界值 - 最小值 50cm
- ✅ 年龄边界值 - 最小值 10岁
- ✅ 目标体重 ≥ 当前体重（targetDate 为 null）

### 5. 邮箱绑定测试（2个）
- ✅ 绑定邮箱
- ✅ 重复绑定同一邮箱

### 6. 订阅与权限测试（3个）
- ✅ 未付费查看结果（脱敏，preview: true）
- ✅ 模拟支付
- ✅ 付费后查看完整结果（含 healthPlan）

### 7. 历史记录测试（2个）
- ✅ 获取历史记录列表
- ✅ 创建第二条记录并验证列表

### 8. 卡路里计算验证（2个）
- ✅ 减重目标应减少卡路里（TDEE - 500）
- ✅ 增肌目标应增加卡路里（TDEE + 400）

**总计：25 个测试用例**

## 运行方式

### 1. 安装依赖
```bash
npm install tsx --save-dev
```

### 2. 启动开发服务器
```bash
npm run dev
```

### 3. 运行测试（新终端）
```bash
npm run test:api
```

## 测试验证的关键点

### 1. 数据完整性
- ✅ 增量保存不丢失之前的数据
- ✅ 提交后 status 改为 completed
- ✅ completed 记录不可修改

### 2. 权限控制
- ✅ 未授权返回 401
- ✅ 越权访问返回 404
- ✅ 禁止操作返回 403

### 3. 业务逻辑
- ✅ BMI 计算正确（70kg / 1.7² ≈ 24.2）
- ✅ 减重目标卡路里 = TDEE - 500
- ✅ 增肌目标卡路里 = TDEE + 400
- ✅ 目标体重 ≥ 当前体重时 targetDate 为 null

### 4. 边界值处理
- ✅ 体重 20-300kg
- ✅ 身高 50-250cm
- ✅ 年龄 10-120岁
- ✅ 所有枚举值（gender/goal/activityLevel）

### 5. 幂等性
- ✅ 重复提交返回已有结果
- ✅ 重复绑定邮箱不报错

### 6. 订阅体系
- ✅ 未付费返回 preview: true
- ✅ 未付费不返回 healthPlan
- ✅ 付费后返回完整数据

## 故障排查

### 测试失败常见原因

1. **服务器未启动**
   ```bash
   npm run dev
   ```

2. **数据库未连接**
   ```bash
   # 检查 .env 文件
   DATABASE_URL="postgresql://..."
   ```

3. **端口冲突**
   - 默认端口 3000
   - 修改 `scripts/test-api.ts` 中的 `BASE_URL`

4. **Prisma Client 未生成**
   ```bash
   npx prisma generate
   ```

## 扩展测试

如需添加新测试，在 `scripts/test-api.ts` 中使用：

```typescript
await test('测试名称', async () => {
  const res = await fetch(`${BASE_URL}/api/xxx`, { ... })
  assert(res.ok, '错误信息')

  const data = await res.json()
  assert(data.field === expectedValue, '字段验证失败')
})
```

## 测试数据清理

测试会创建多条记录，可通过以下方式清理：

```sql
-- 删除测试邮箱的用户
DELETE FROM users WHERE email LIKE 'test-%@example.com';

-- 或使用定时清理接口
GET /api/cron/cleanup
```
